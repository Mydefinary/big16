## Backend 로그 분석
Backend가 정상 실행되고 있으며 FastAPI 서버가 8083 포트에서 동작 중:
- "POST /question-api/ask HTTP/1.1" 404 Not Found 
- "GET /question-api/health HTTP/1.1" 404 Not Found

## 문제점 식별
Backend 로그에서 404 에러가 발생하는 것으로 보아, Backend 서비스는 `/question-api/` prefix 없이 엔드포인트가 구성되어 있음:
- Gateway: /question-api/** → Backend
- Backend 실제: /ask, /health (prefix 없음)
- 요청 경로: /question-api/ask → Backend에서 404

## 해결 방안
1. Frontend API 요청을 /question-api/ask 대신 /question-api/로 변경 (완료)
2. 이렇게 하면 Gateway가 /question-api/를 제거하고 Backend의 /ask로 라우팅

- Backend 엔드포인트 구조 확인
