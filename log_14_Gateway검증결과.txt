## 검증 방법 및 증거

### 1. Backend 라우팅 검증
**테스트**: `curl http://20.249.113.18:9000/question-api/health`
**결과**: 500 Internal Server Error
**메시지**: `Connection refused: question-backend-service-lee-2/10.0.214.86:8083`

**분석**: Gateway가 정확히 `question-backend-service-lee-2:8083`로 라우팅하고 있음

### 2. Frontend 라우팅 검증  
**테스트**: `curl http://20.249.113.18:9000/question/`
**결과**: 200 OK, HTML 응답
**서버**: nginx/1.29.1

**분석**: Gateway가 `question-frontend-service-lee-2:80`로 정상 라우팅

### 3. 서비스 매핑 상태
```
Gateway 라우팅      →  Kubernetes Service      →  Pod 상태
question-api/**     →  question-backend-lee-2   →  CrashLoopBackOff ❌
question/**         →  question-frontend-lee-2  →  Running ✅
```

## 현재 문제점

### 정확한 원인
- **Gateway 설정**: 정상 ✅
- **Frontend 서비스**: 정상 동작 ✅  
- **Backend 서비스**: CrashLoopBackOff ❌

### 에러 분석
Gateway 에러 메시지에서 확인:
- IP: 10.0.214.86 (question-backend-service-lee-2의 ClusterIP)
- Port: 8083 (정확한 포트)
- 문제: Backend pod가 시작되지 않아 Connection refused

## 해결 방안

### 1. Backend Pod 수정 (진행 중)
- chatbot backend를 question-backend-deployment-lee-2에 배포
- Docker 이미지 빌드 진행 중

### 2. 환경 변수 확인 필요
- OPENAI_API_KEY가 올바르게 설정되어 있는지 확인
- 기타 필요한 환경 변수들 점검

### 3. 최종 목표
- `/question-api/**` 경로에서 chatbot API 정상 동작
- `/question/**` 경로에서 chatbot frontend 정상 동작
